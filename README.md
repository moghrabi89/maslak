# مسلك - درب الشافعي 🧭

منصة تعليمية تفاعلية شاملة لدراسة الفقه الشافعي بأسلوب التلعيب العلمي (Gamified Learning)، مستوحاة من نمط Duolingo، مع نظام حوكمة فقهية صارم ومولد للمسائل الفقهية ومراجعة متباعدة ذكية.

---

## 🚀 التقنيات المستخدمة (Tech Stack)

* **الإطار البرمجي:** Next.js 16 (App Router)
* **لغة البرمجة:** TypeScript (صارم)
* **التصميم والواجهات:** Tailwind CSS v4 & Hero UI & Framer Motion
* **قاعدة البيانات:** PostgreSQL (Neon Serverless)
* **مخطط الكود وقارئ البيانات (ORM):** Drizzle ORM
* **المصادقة والمستخدمين:** Clerk
* **إدارة الملفات والصور:** UploadThing
* **التحقق من البيانات:** Zod

---

## 📁 هيكلية مجلدات المشروع (Directory Structure)

* `app/`: صفحات المنصة ومساراتها بالاعتماد على App Router (لوحة الطالب، لوحة الإدارة، شاشات المذاكرة والتحديات).
* `components/`: المكونات المشتركة للواجهة (البطاقات، النوافذ العائمة، أزرار عرض المصادر، Mascot).
* `db/`: إعدادات الاتصال بقاعدة البيانات (`index.ts`) وتصميم الجداول الفقهية السبعة عشر وعلاقاتها (`schema.ts`).
* `lib/`: ملفات التحقق والدوال المساعدة المشتركة (مثل التحقق من المتغيرات البيئية `env.ts`).
* `actions/`: دوال Server Actions لإجراء العمليات الحسابية والمنطق الفقهي والخلفي.
* `data/`: نصوص التغذية الأولية (Seeding) والبيانات الثابتة للمنصة.
* `hooks/`: خطافات React المخصصة لتسهيل عمليات الواجهة.
* `types/`: تعريفات الأنواع (Types) الفقهية والطلابية المخصصة.
* `tests/`: اختبارات الوحدة والاختبارات الآلية للتحقق من مولد الأسئلة وموثوقية التعليم.

## ⚙️ التجهيز والتشغيل المحلي (Setup & Run)

### 1. تثبيت الاعتمادات
```bash
npm install
```

### 2. إعداد المتغيرات البيئية
أنشئ ملف `.env` في المجلد الرئيسي واملأ المتغيرات التالية:
```env
# اتصال قاعدة البيانات (Neon PostgreSQL)
DATABASE_URL="postgresql://..."

# مصادقة المستخدمين (Clerk)
NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY="pk_..."
CLERK_SECRET_KEY="sk_..."

# إدارة الملفات (UploadThing)
UPLOADTHING_SECRET="sk_..."
UPLOADTHING_APP_ID="..."
```

### 3. إنشاء قاعدة البيانات والمهاجرة (Migrations)
لتحديث مخطط قاعدة البيانات محليًا وإنشاء الجداول:
```bash
npx drizzle-kit generate
npx drizzle-kit push
```

### 4. تشغيل خادم التطوير محليًا
```bash
npm run dev
```

## ⚖️ مسار الحوكمة الشرعية والعلمية للمحتوى

تعتمد المنصة دورة حياة صارمة للمحتوى لضمان عدم ظهور أي مسألة فقهية غير معتمدة علميًا للطلاب:

$$\text{Draft} \longrightarrow \text{Reviewed} \longrightarrow \text{Approved} \longrightarrow \text{Published} \longrightarrow \text{Archived}$$

1. **Draft:** مسودة مبدئية يضيفها كاتب المحتوى.
2. **Reviewed:** تمت مراجعة المادة لغويًا وفنيًا.
3. **Approved:** اعتمدها المحقق العلمي الفقهي ورُفع مستوى الثقة العلمية إلى 100%.
4. **Published:** منشورة تظهر للطلاب في مسارات شجرة التعلم.
5. **Archived:** مؤرشفة (تُخفى عن الطلاب الجدد مع الاحتفاظ بتقدم الطلاب القدامى لضمان سلامة قاعدة البيانات).
